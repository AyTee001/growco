# Робота з репозиторієм

Щоб уникнути конфліктів у коді та зберегти стабільність проєкту, у цьому репозиторії діють наступні правила.

---

## 🔀 Робота тільки через Pull Request

Усі зміни вносяться до основної гілки master **виключно через Pull Request**.

Типовий процес роботи:

1. Створіть нову гілку від `master`
2. Виконайте зміни у своєму коді
3. Зробіть `push` у свою гілку
4. Створіть **Pull Request**
5. Після перевірки зміни будуть об'єднані з `master`

## 🌿 Назви гілок

Назва гілки повинна коротко описувати зміни і зазначати ідентифікатор завдання (якщо є пов'язане завдання)

## 🚫 Заборонено пушити напряму в `master`

Усі зміни повинні проходити через **Pull Request**.

## 🛠 Проблеми з доступом

Якщо у вас:

- немає доступу до репозиторію
- неможливо створити гілку
- не виходить запушити зміни
- виникли проблеми з Pull Request чи треба його перевірити

зверніться до **відповідальних за репозиторій**
